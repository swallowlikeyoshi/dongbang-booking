# 동방 예약 시스템 — 설계 문서

작성일: 2026-07-21
상태: 설계 확정 (구현 계획 대기)

## 1. 목적

HEVEN 동아리방 2개를 팀원들이 시간 슬롯 단위로 예약하는 웹 페이지. 구글 로그인으로 신원을 확인하고, 관리자는 별도 페이지에서 전체 예약을 운영한다. 젯슨 나노에서 도커로 24시간 구동한다.

기존 `luftaquila/ksae-notice` repo의 검증된 스택(Next.js + SQLite + Auth.js + Docker)을 재사용한다. 범용 예약 오픈소스(MRBS 등)는 LAMP 스택·ARM 배포 불확실·OAuth 패턴 불일치로 우리 요구(리소스 2개, 규칙 없음)에 과잉이라 채택하지 않았다.

## 2. 핵심 요구사항

- **리소스**: 동아리방 2개, 각각 독립 예약.
- **예약 단위**: 시간 슬롯, **30분 격자**.
- **로그인**: 아무 구글 계정이나 로그인 가능(신원 확인용). 화이트리스트·도메인 제한 없음.
- **열람**: 로그인 없이도 캘린더 현황판을 **읽기 전용**으로 볼 수 있다. 예약 생성/취소는 로그인 필요.
- **규칙**: 없음. 빈 슬롯은 누구나 잡고 본인이 취소. 단, 같은 방 시간 겹침은 서버가 거절(데이터 정합성).
- **팀 구분**: 예약마다 `전기팀` / `기계팀` / `자율차팀` / `기타` 선택. 캘린더에서 팀별 색상 구분.
- **설명**: 예약마다 간단한 일정 이름(`title`), 선택 입력(비워도 됨).
- **관리자**: `ADMIN_EMAIL` 환경변수에 포함된 계정만 관리자 페이지 접근. 전체 예약 열람, 강제 취소, 방 이름 수정.

## 3. 기술 스택

| 계층 | 기술 |
|------|------|
| 프레임워크 | Next.js (App Router, TypeScript) — 프론트 + API 라우트 한 몸 |
| DB | SQLite + Drizzle ORM (파일 하나, 볼륨 마운트로 영속화) |
| 인증 | Auth.js (NextAuth) + Google Provider |
| 배포 | Containerfile → 젯슨 나노 도커 24시간 구동 |
| CI/CD | GitHub Actions → GHCR 자동 빌드/푸시 (무료 범위) |

## 4. 데이터 모델

### `rooms`
방은 2개 고정이지만 이름 변경 편의를 위해 테이블로 둔다.

| 필드 | 타입 | 설명 |
|------|------|------|
| id | integer PK | 1, 2 |
| name | text | "동방 A", "동방 B" (관리자 수정 가능) |

### `reservations`

| 필드 | 타입 | 설명 |
|------|------|------|
| id | integer PK | |
| room_id | integer FK → rooms.id | 어느 방 |
| team | text | `전기팀` / `기계팀` / `자율차팀` / `기타` |
| title | text nullable | 간단한 일정 이름 (선택 입력) |
| user_email | text | 예약자 구글 계정 |
| user_name | text | 표시용 이름 |
| start_at | integer (unix ts) | 시작 시각, 30분 격자에 스냅 |
| end_at | integer (unix ts) | 종료 시각, 30분 격자에 스냅 |
| created_at | integer (unix ts) | 생성 시각 |

- **취소**: 행 삭제(soft-delete는 불필요, YAGNI).
- **겹침 검사**: 생성 시 서버에서 `room_id` 동일 + 시간 구간 교차 여부 확인 → 교차 시 거절.
- **권한**: 예약 취소는 `user_email == 세션 이메일` 이거나 관리자일 때만.

## 5. 팀 색상 규칙

| 팀 | 색 |
|------|------|
| 전기팀 | 파랑 |
| 기계팀 | 주황 |
| 자율차팀 | 초록 |
| 기타 | 회색 |

## 6. 화면 구성

1. **메인 (주간 캘린더)** — 비로그인 포함 누구나 접근. 방 2개(나란히 또는 탭), 세로축 = 시간(30분 격자), 가로축 = 요일(월~일). 예약 블록은 팀 색상 + `title` 표시. 주 이동(이전/다음 주).
2. **예약 만들기** — 로그인 상태에서 빈 슬롯 클릭/드래그 → 팝업(방/팀/설명/시작·종료 확인) → 저장. 겹치면 서버 거절 메시지.
3. **내 예약** — 본인 예약 목록 + 취소 버튼.
4. **관리자 페이지** — `ADMIN_EMAIL` 계정만. 전체 예약 열람, 아무 예약이나 강제 취소, 방 이름 수정.

## 7. 예약 흐름

```
구글 로그인 → 주간 캘린더 → 빈 슬롯 선택 → 팀/설명 입력 → 저장(겹침 검사 통과) → 캘린더 즉시 반영
취소: 본인 예약(내 예약 페이지) 또는 관리자(관리자 페이지)
```

## 8. 인증 · 권한

- Auth.js Google Provider. 세션에 `email`, `name` 보관.
- 관리자 판정: 세션 `email`이 `ADMIN_EMAIL`(쉼표 구분 다중) 목록에 포함되면 관리자.
- API 라우트에서 서버측 재검증(생성/취소/관리 액션 모두). 클라이언트 표시만으로 권한을 신뢰하지 않는다.
- 읽기(캘린더 조회) API는 비인증 허용.

## 9. 배포

- Containerfile로 이미지 빌드 → GitHub Actions가 `main` 푸시 시 GHCR로 푸시.
- 젯슨에서 도커로 pull & run. SQLite 파일은 볼륨 마운트.
- 환경변수:
  - `AUTH_GOOGLE_ID`, `AUTH_GOOGLE_SECRET`
  - `AUTH_SECRET` (`npx auth secret`)
  - `ADMIN_EMAIL` (쉼표로 다중 관리자)
  - `NEXTAUTH_URL` (젯슨 노출 도메인 — ksae-notice와 동일한 Tailscale Funnel 방식 등)
- 구글 OAuth 콘솔에 redirect URI 등록(`{도메인}/api/auth/callback/google`).

## 10. 범위 밖 (YAGNI)

- 예약 규칙(1인당 시간 제한, 선예약 기간 제한, 당일취소 금지 등) — 요청 시 후속.
- 반복 예약, 통계/대시보드, 이메일 알림.
- 화이트리스트/도메인 제한 로그인.
- 방 개수 동적 추가(2개 고정).

## 11. 열린 소소한 결정 (구현 중 확정)

- 방 2개 표시: 나란히 2열 vs 탭 전환 — 화면 폭 보고 결정.
- 주간 캘린더 라이브러리: 직접 CSS 그리드 구현 vs 경량 캘린더 컴포넌트 — 구현 계획에서 선택.
